## Purpose

This appendix defines the core business intelligence (BI) and decision-science terminology used throughout Volume 04. It establishes a shared vocabulary for humans and the AI Business Partner so that reports, dashboards, and decision records use terms consistently.

## Scope

The glossary covers analytics, data management, decision science, and performance-management terms referenced in Volume 04. Definitions are concise operational definitions, not exhaustive academic treatments.

## Glossary

| Term | Definition |
|---|---|
| A/B Test | A controlled experiment comparing two variants (A and B) to determine which performs better on a target metric. |
| Actionable Insight | An analytical finding specific and clear enough to drive a concrete decision or action. |
| Aggregation | The process of summarizing granular data into higher-level totals (e.g., sum, average, count). |
| Anomaly Detection | Identifying data points that deviate significantly from expected patterns. |
| Baseline | A reference measurement against which change or performance is compared. |
| Benchmark | A standard or peer reference point used to evaluate relative performance. |
| Business Intelligence (BI) | The technologies, processes, and practices used to collect, integrate, analyze, and present business data to support decisions. |
| Cohort Analysis | Grouping subjects sharing a common characteristic or time window and tracking their behavior over time. |
| Confidence Interval | A range of values that is likely to contain a population parameter at a stated probability level. |
| Correlation | A statistical measure of how two variables move together; does not imply causation. |
| Dashboard | A visual interface consolidating key metrics and indicators for at-a-glance monitoring. |
| Data Governance | The framework of policies, roles, and standards ensuring data quality, security, and proper use. |
| Data Lineage | The documented path of data from source through transformations to consumption. |
| Data Mart | A subject-specific subset of a data warehouse serving a particular team or function. |
| Data Warehouse | A centralized repository of integrated, structured data optimized for analysis and reporting. |
| Descriptive Analytics | Analytics that summarize what happened using historical data. |
| Diagnostic Analytics | Analytics that explain why something happened by identifying causes and relationships. |
| DIKW Pyramid | The Data-Information-Knowledge-Wisdom hierarchy describing how raw data becomes actionable wisdom. |
| Dimension | A qualitative attribute used to categorize and filter facts (e.g., region, product, time). |
| Drill-Down | Navigating from summary data to more detailed underlying data. |
| Driver | A variable that materially influences an outcome metric (e.g., price is a revenue driver). |
| ETL / ELT | Extract, Transform, Load (or Extract, Load, Transform); processes for moving and preparing data for analysis. |
| Expected Value (EV) | The probability-weighted average of all possible outcomes of a decision. |
| Forecast | A projection of future values based on historical data and assumptions. |
| Granularity | The level of detail represented in a data set (e.g., daily vs. monthly). |
| Hypothesis | A testable proposition about a relationship or outcome, evaluated with data. |
| Key Performance Indicator (KPI) | A quantifiable metric tied to a strategic or operational objective. |
| Key Result | A measurable outcome that indicates progress toward an objective (see OKR). |
| Lagging Indicator | A metric that reflects past performance and outcomes (e.g., revenue). |
| Leading Indicator | A metric that predicts future performance (e.g., pipeline coverage). |
| Materiality | The threshold above which an item is significant enough to affect a decision. |
| Median | The middle value of an ordered data set; robust to outliers. |
| Metric | A quantified measure used to track and assess a process or outcome. |
| Normalization | Adjusting values to a common scale to enable fair comparison. |
| North Star Metric | The single metric that best captures the core value delivered to customers. |
| OKR | Objectives and Key Results; a goal-setting framework linking ambitions to measurable outcomes. |
| Outlier | A data point markedly distant from the rest of a distribution. |
| Predictive Analytics | Analytics that estimate what is likely to happen using statistical or machine-learning models. |
| Prescriptive Analytics | Analytics that recommend actions to achieve desired outcomes. |
| Regression | A statistical technique modeling the relationship between a dependent variable and predictors. |
| RFM Analysis | Segmentation by Recency, Frequency, and Monetary value of customer transactions. |
| Sample | A subset of a population selected for analysis. |
| Scorecard | A structured summary of performance against targets across defined perspectives. |
| Segmentation | Dividing a population into meaningful, distinct groups for targeted analysis. |
| Sensitivity Analysis | Testing how changes in inputs affect an output or decision outcome. |
| Single Source of Truth (SSOT) | An authoritative, agreed data source used for a given metric or entity. |
| Statistical Significance | The likelihood that an observed result is not due to random chance. |
| Time Series | A sequence of data points indexed in time order. |
| Trend | The general direction of a metric over time. |
| Variance | The difference between actual and expected (or budgeted) values; also, a statistical dispersion measure. |
| Weighted Average | An average in which each value contributes proportionally to an assigned weight. |
