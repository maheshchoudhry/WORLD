## Purpose

This appendix is a working catalog of the analytical models and methods applied throughout Volume 04. For each model it documents purpose, required inputs, produced outputs, and guidance on when to use it, so the AI Business Partner can select and apply the right method to a given problem.

## Scope

The catalog covers strategic analysis, root-cause analysis, financial evaluation, decision analysis, customer analytics, and forecasting methods. Each entry is a reference summary; detailed procedures live in the corresponding Volume 04 chapters.

## Model Catalog

### 1. SWOT Analysis

| Attribute | Detail |
|---|---|
| Purpose | Assess internal Strengths and Weaknesses against external Opportunities and Threats to inform strategy. |
| Inputs | Internal capability data, market intelligence, competitor and macro-environment scan. |
| Outputs | A four-quadrant matrix and prioritized strategic implications (SO, WO, ST, WT actions). |
| When to use | Early-stage strategy formulation, annual planning, business unit or initiative review. |

### 2. Porter's Five Forces

| Attribute | Detail |
|---|---|
| Purpose | Evaluate industry attractiveness and competitive intensity. |
| Inputs | Supplier landscape, buyer concentration, entry barriers, substitute availability, rivalry data. |
| Outputs | Force-by-force intensity ratings and an overall assessment of structural profitability. |
| When to use | Market-entry decisions, competitive positioning, industry attractiveness assessment. |

### 3. Value Chain Analysis

| Attribute | Detail |
|---|---|
| Purpose | Identify where value is created and where cost or differentiation advantage can be built. |
| Inputs | Primary activities (inbound logistics, operations, outbound logistics, marketing/sales, service) and support activities with cost and margin data. |
| Outputs | Activity-level cost/value map and improvement or differentiation opportunities. |
| When to use | Operating-model design, cost optimization, competitive advantage analysis. |

### 4. 5 Whys

| Attribute | Detail |
|---|---|
| Purpose | Trace a problem to its root cause through iterative questioning. |
| Inputs | A clearly stated problem and factual observations at each causal layer. |
| Outputs | A causal chain terminating in one or more root causes and corrective actions. |
| When to use | Straightforward, single-thread problems; quick root-cause investigation. |

### 5. Ishikawa (Fishbone) Diagram

| Attribute | Detail |
|---|---|
| Purpose | Organize potential causes of a problem into categories to structure diagnosis. |
| Inputs | Problem statement and candidate causes grouped by category (e.g., People, Process, Technology, Materials, Environment, Measurement). |
| Outputs | A cause-and-effect diagram highlighting probable root-cause categories. |
| When to use | Complex, multi-factor problems requiring team brainstorming and structured grouping. |

### 6. Theory of Constraints (TOC)

| Attribute | Detail |
|---|---|
| Purpose | Maximize throughput by identifying and managing the system's binding constraint. |
| Inputs | Process flow, capacity by step, throughput, inventory, and operating expense data. |
| Outputs | Identified bottleneck plus the Five Focusing Steps (identify, exploit, subordinate, elevate, repeat). |
| When to use | Operations and process optimization where a bottleneck limits output. |

### 7. Net Present Value (NPV)

| Attribute | Detail |
|---|---|
| Purpose | Value an investment by discounting future cash flows to present value. |
| Inputs | Projected cash flows by period, discount rate (cost of capital), initial outlay. |
| Outputs | NPV figure; positive NPV indicates value creation. Companion metrics: IRR, payback period. |
| When to use | Capital allocation, investment appraisal, project prioritization. |

### 8. Expected Value (EV)

| Attribute | Detail |
|---|---|
| Purpose | Evaluate decisions under uncertainty by probability-weighting outcomes. |
| Inputs | Possible outcomes, their payoffs, and estimated probabilities (summing to 1). |
| Outputs | A single expected-value figure per option; used with decision trees. |
| When to use | Risk-bearing choices, scenario comparison, decision-tree analysis. |

### 9. RFM Analysis

| Attribute | Detail |
|---|---|
| Purpose | Segment customers by Recency, Frequency, and Monetary value to target action. |
| Inputs | Transaction history with dates, counts, and values per customer. |
| Outputs | RFM scores and segments (e.g., champions, at-risk, lapsed) with recommended treatments. |
| When to use | Customer retention, marketing targeting, lifetime-value management. |

### 10. Time-Series Forecasting

| Attribute | Detail |
|---|---|
| Purpose | Project future values from historical, time-ordered data. |
| Inputs | Historical series with sufficient history; identification of trend, seasonality, and cycles. |
| Outputs | Point forecasts with confidence intervals (e.g., moving average, exponential smoothing, ARIMA). |
| When to use | Demand planning, revenue projection, capacity planning when history is stable. |

### 11. Driver-Based Forecasting

| Attribute | Detail |
|---|---|
| Purpose | Model an outcome as a function of its underlying operational drivers. |
| Inputs | Driver values (e.g., traffic, conversion, price, headcount) and their relationships to the target. |
| Outputs | Forecasts that flex with driver assumptions; enables scenario and sensitivity analysis. |
| When to use | Budgeting, planning, and what-if analysis where causal drivers are known and controllable. |
