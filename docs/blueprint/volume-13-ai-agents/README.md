## Purpose
Define the complete AI Agent architecture of Project WORLD: agent philosophy, lifecycle, framework, runtime and identity, cognition, collaboration and control, the catalog of core and specialist agents, and agent governance. This volume operationalizes the AI Business Partner (Volume 03) as concrete, governed agents that work inside the ERP (Volumes 05-06), reason over the Knowledge Engine (Volume 14), and run on the architecture and infrastructure (Volumes 08-12).

## Scope
Agent foundations; runtime and identity; cognition (memory, tools, knowledge, planning, reasoning, reflection, learning); collaboration and control (communication, orchestration, human approval, multi-agent); the core and specialist agent catalog; and governance and evolution, plus appendices. Every agent operates under the human-in-the-loop and governance rules of Volume 03 (Section G) and the security architecture of Volume 12.

## How to Use This Volume
Framework chapters follow the standard architecture template (Concept, Architecture, Key Components, Trade-offs, Relationships). Agent chapters (Sections E and F) follow the agent-definition template: Responsibilities, Capabilities, Inputs, Outputs, Tools, Knowledge Sources, Decision Authority, Human Approval Requirements, KPIs, and Security Boundaries. All chapters use Mermaid diagrams, tables, examples, and cross-references.
